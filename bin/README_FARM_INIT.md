# Farm Initialization Scripts

## Overview

Two versions of the farm initialization script are available:

1. **`farm_init.py`** - Original version (has compatibility issues)
2. **`farm_init_v2.py`** - Improved robust version (recommended)

## Recommended: Use farm_init_v2.py

### Quick Start
```bash
cd /opt/mining-stack/bin
python3 farm_init_v2.py
```

### What It Does
1. **Scans network** for devices with open mining ports (4028, 80, 8080)
2. **Identifies miners** using PyASIC library
3. **Generates configuration** aligned with collector requirements
4. **Validates output** to ensure correctness
5. **Backs up** existing config automatically

## Key Improvements in v2

### 1. Correct Output Format
✅ Flat structure compatible with all collectors
✅ All required fields present (`ip`, `name`, `model`)
✅ Credentials in correct format (`username`, `password`)

### 2. Smart Name Generation
```python
"Antminer S19j Pro" + "192.168.1.100" → "s19j-100"
"Whatsminer M30S+" + "192.168.1.101"  → "m30sp-101"
"ElphaPex DG1" + "192.168.1.102"      → "dg1-102"
```

### 3. Model-Based Credentials
- **Whatsminer**: `admin/admin`
- **Antminer**: `root/root`
- **Others**: `root/root` (default)

### 4. Robust Error Handling
- Continues on individual failures
- Reports partial results
- Graceful keyboard interrupt (Ctrl+C)

### 5. Configuration Validation
- Checks required fields
- Detects duplicate names/IPs
- Validates IP addresses

### 6. Auto-Backup
- Backs up existing config before overwriting
- Creates `.backup` file automatically

## Troubleshooting

### No Miners Found
**Possible causes**:
- Miners are offline
- Miners are on different network
- Firewall blocking connections

**Solutions**:
- Check miner power and network
- Verify network configuration
- Add IPs manually to `EXTRA_IPS` in script

### Identification Fails
**Possible causes**:
- PyASIC doesn't support model
- Miner API not responding
- Network timeout

**Solutions**:
- Check miner web interface
- Verify API is enabled
- Increase timeout in script

### Permission Denied
**Solution**:
```bash
sudo python3 farm_init_v2.py
# or
sudo chown $USER /opt/mining-stack/etc/
```

## Migration from Original

### Step 1: Backup Current Config
```bash
cp /opt/mining-stack/etc/miners.yaml /opt/mining-stack/etc/miners.yaml.old
```

### Step 2: Run New Version
```bash
python3 /opt/mining-stack/bin/farm_init_v2.py
```

### Step 3: Review Output
```bash
cat /opt/mining-stack/etc/miners.yaml
```

### Step 4: Test with Collectors
```bash
docker restart python-scheduler
docker logs python-scheduler -f
```
